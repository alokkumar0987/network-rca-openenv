Deploy to Hugging Face Space: alok098/network-rca-env (Docker SDK)
==================================================================

SECURITY (do this first)
------------------------
You pasted a Hugging Face token in chat / .env. Treat it as compromised.

1. Open https://huggingface.co/settings/tokens
2. Revoke/delete that token.
3. Create a NEW token with "Write" access (for git push to Spaces).

Never commit tokens. Use Space "Settings → Variables and secrets" for runtime keys.


Where your app code lives
-------------------------
This repo folder (network-rca-openenv) is the full OpenEnv project to deploy.
(Parent folder D:\project\network-rca-env may only hold docs; deploy FROM here.)


One-time: install Hugging Face CLI (PowerShell)
-----------------------------------------------
powershell -ExecutionPolicy Bypass -c "irm https://hf.co/cli/install.ps1 | iex"

Then login (uses secure storage; do not paste token into .env):
huggingface-cli login


Option A — Git push (recommended)
---------------------------------
1) Clone the empty Space repo (use your NEW token when Git asks for password):

   git clone https://huggingface.co/spaces/alok098/network-rca-env
   cd network-rca-env

2) Copy all files from this folder (network-rca-openenv) into the clone
   (Dockerfile, app.py, requirements.txt, data/, tasks.py, etc.).

3) Commit and push:

   git add -A
   git commit -m "Deploy Network RCA OpenEnv"
   git push

Git credential hint: username can be your HF username; password = the token.


Option B — huggingface_hub from Python (alternative)
--------------------------------------------------
See: https://huggingface.co/docs/hub/spaces-sdks-docker


Required Space settings
-----------------------
1) SDK: Docker (you already chose Docker Space).

2) App must listen on port 7860 (Dockerfile CMD already uses 7860).

3) Secrets (Settings → Variables and secrets):
   - OPENAI_API_KEY = your OpenAI key (for baseline / LLM in container)

   Optional if downloads hit rate limits:
   - HF_TOKEN = a read token (for model hub downloads inside the container)


After push
----------
Wait for the Space build log. Fix errors from "Logs" tab.
Smoke test when green:
   GET https://YOUR_SPACE_URL/
   POST /reset with JSON {"difficulty":"easy"}


OpenEnv tag
-----------
If the hackathon requires the Space to be tagged "openenv", set that in
Space metadata / tags in the Space settings or README as per their checklist.
